# The IE Field Guide

> A Practical Guide to Studying Information Engineering

**The IE Field Guide** is an English-first, version-controlled guidebook for Information Engineering students.

It is written in Markdown, maintained with Git, and distributed primarily as PDF.

This project was originally nicknamed **“IE 天书”** — a student-written guide meant to be passed around, revised, and rediscovered by future students.

## Project Philosophy

This guide is not a textbook.

It is not meant to replace lectures, official course materials, academic advising, or formal curriculum documents. Instead, it aims to provide a practical map for students who are trying to understand Information Engineering, survive the early years of university, build technical foundations, and make better long-term decisions.

A field guide is written for people navigating a real environment. That is the spirit of this project.

## Core Principles

- English first.
- Markdown as the source format.
- Git for version control.
- PDF as the primary distribution format.
- AI-assisted, human-directed writing.
- Pseudonymous public authorship.
- No PDF, no release.

## AI-Assisted Writing Disclosure

This guide is written with AI assistance and human editorial control.

AI tools may be used for brainstorming, outlining, rewriting, proofreading, restructuring, language polishing, consistency checking, and improving readability. All published content should be reviewed, selected, and approved by the human author.

The purpose of using AI is not to replace personal reflection, but to support clearer writing and long-term revision.

## Target Readers

The primary readers are:

- Information Engineering students
- Incoming freshmen
- Students preparing for programming and CS-related courses
- Students trying to understand projects, research, internships, and career paths
- Students looking for practical orientation beyond official course descriptions

## Distribution Format

PDF is the required release format.

Markdown files are used for writing and version control. PDF files are used for official distribution, especially in student communities, OCamp settings, major group chats, Telegram channels, WhatsApp groups, and other informal circulation channels.

In short:

- Markdown is the source format.
- PDF is the release format.
- Git is the memory of the project.
